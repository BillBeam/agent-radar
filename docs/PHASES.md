# Agent Radar — 阶段与步骤说明（大白话留痕）

> 这份文档用**大白话**记录每个阶段（P0–P4）做了哪些步、每一步在解决什么问题、为什么这么做。
> 它是「过程留痕」，方便回头看；严谨的技术真相见 [SPEC.md](SPEC.md)，需求与协作方式见 [REQUIREMENTS.md](REQUIREMENTS.md)。

## 这个 agent 到底在干嘛（一句话）

每天自动抓全球最前沿的 agent/harness 技术文章 → AI 读懂、写成**中文详解** → 推给我 + 本地存档。
目标不是「能跑就行」，而是**干净、可信、对我精确、够深、诚实、记得历史、越用越准、能对话**——分五个阶段（P0–P4）一步步逼近。

## P0 地基 —— 把「能跑」做成「靠谱又干净」

### 0. 每日管线本身（底座，最先有）

一条流水线，7 道工序，像工厂的传送带：

| 工序 | 大白话 | 用什么 |
|---|---|---|
| **抓取 fetch** | 从 ~28 个源拉文章、去重 | 纯代码 |
| **分诊 triage** | 给一大池候选快速打标签、滤掉明显垃圾 | 便宜模型 haiku |
| **质量门 quality_gate** | 按规则砍噪音、裁到「决赛圈」（约 24 篇） | 纯代码规则 |
| **重排 rerank** | 给决赛圈**排名次**（谁更重要） | 中档模型 sonnet |
| **深读 deepread** | 对最终选中的约 6 篇，只读原文、写中文详解 | 强模型 opus |
| **合成 synthesize** | 拼成两版：完整版（本地）+ 精简版（钉钉） | 纯代码 |
| **投递 deliver** | 推钉钉 + 本地存档 | 纯代码 |

底座能跑之后，发现真实产出「又丑、又没区分度、又有静默 bug」，于是做了 5 块加固（**D/C/B/A/E**）：

### D · 可达性 —— 「先保证能抓到西方网站」
我在国内、源大多是西方站点。修了个真 bug：之前代码会把我系统挂的代理**一刀切关掉**，导致明明挂了代理却抓不到。改成**尊重环境里的代理**；再加 `doctor` 体检，实测几个代表源通不通、延迟多少，全失败就大声告警。

### C · 可信 —— 「坏了要吱声，别假装没事」
堵了几个静默坑：①全网抓失败它还输出「今天没料，明天见」——我以为没料、其实是断网 → 改成**全失败会告警**、并区分「真没料」vs「抓取挂了」。②分诊只返回一半结果就悄悄缩水 → **缺的兜底补上、并记录覆盖率**，低了告警。③每次跑完写一份 `last_run` 摘要（含 token 用量），`status` 能查。④加**运行锁**，防两个进程同时跑、抢文件、重复投递。

### B · 选择 —— 「排得出高下，别全是 9 分」
两个问题：①分诊给**绝对分**容易全挤在 9 分，没区分度 → 新增**重排**工序，让模型对决赛圈**相对排名次**，逼出梯度（10→2 这种）。②无日期的博客索引页会把**整个历史**当「今天」灌进来 → 分成 **🆕今日新增 / 📚往期补课** 两区，旧文不当今日；再加**多样性配额**，防单个源（比如某大厂博客）刷屏。

### A · 呈现 —— 「钉钉上看着不丑、能一眼扫」
钉钉的 markdown 不渲染反引号之类，之前卡片很脏。改成干净可扫的卡片：**去掉反引号、「相关度 N」、星标**，清掉标题尾巴的日期，详解小标题改加粗行（不和外层抢层级），不在词中间截断。

### E · 反馈种子 —— 「我能给每条点赞/踩，为以后个性化攒数据」
加了 `radar mark <日期> <编号> --up/--down`。关键是修了一个会**静默标错条目**的坑：让「我看到的顺序、存档顺序、点赞编号」三者永远对齐。点赞会连**内容快照**一起存，给 P2 个性化用。

> **P0 状态：D/C/B/A/E 全部完成、已推送，端到端真跑验证过。**

## P1 尺子 —— 「给系统装上能测量好坏的标尺」

**为什么先做这个**：现在改 AI 的提示词/模型，到底变好变坏，全靠肉眼。装上尺子后——**改完跑一遍，分涨了才留、跌了回退**，不再凭感觉。它也是后面「个性化」「自我进化」的判据：没尺子，自我进化就是自吹。
入口是 `radar --mode eval <日期>`。**从 2026-07-05 起尺子自动运转**：它接在每天 daily 之后自动跑（eval 失败只记日志、绝不影响已完成的投递；撞额度靠逐篇存盘次日续上），每周日 `radar --mode review` 再自动盘点读数、把摘要推到钉钉——**「跑」和「送达」全自动，要不要改、怎么改仍由我拍板**。手动随时可跑。

### ① 忠实度 eval + 原文留存（✅ 已完成）
测**详解有没有忠于原文、还是 AI 在瞎编**：
- 顺带给深读加个小动作：把「AI 写详解时**实际看的那份原文**」单独存一份（sidecar），量得最准；历史的也能用当时存下的正文兜底。
- 请**另一个 AI 当审计员**：把详解拆成一句句「事实主张」，逐句回原文找证据——**有据可查 / 原文没有 / 说反了**。
- **用程序算分**：合格占几成 = 忠实度分。让 **AI 负责判断、程序负责算数**——分才稳、能一天天对比，也不容易被「写得专业」糊弄高分。
- **诚实**：没原文/没深读的篇会跳过，报告**必标覆盖率**（评了几篇 / 跳了几篇），不拿「均分」糊弄。
- 今天还顺手把它打磨成**怎么都不亏**：失败会说清原因（限流/超时/解析）、超时只试一次不重试白烧、**判完一篇即存盘可续跑**。

### ② 排序 eval（✅ 已完成）
测**名次排得对不对**：
- **主看我的反馈**：我点赞的有没有排在点踩的前面（成对准确率）。反馈还很少，会**如实报样本量**，0 条就直说「暂无足够反馈」，不装有信号。
- **次看独立裁判**：另一个 AI 按「重要 + 对我新」重排一遍，和系统名次比一致度，给一个**不依赖反馈**的参考。

### ③ 报告 + 醒目结论（✅ 已完成）
把两个维度拼成一份**能跨天对比**的报告：一眼看到的结论行（忠实度均值 X%、覆盖率、排序准确率），存成结构化文件，以后改系统就靠它看数字有没有涨。

## P2–P4 —— 现状（2026-07-05 照实更新）

- **P2 懂你（✅ 已完成）**：记忆（SQLite FTS5 + USER.md，不向量）+ 个性化——手填「已会清单」接进 rerank 给已会主题降权，推「对我重要**且对我新**」的。真跑 A/B 验收过：已会主题（harness 构建 / brain-hands 解耦这类）沉下去、真前沿不被误杀。**而且是在 P0 隐私修复之后第一次「干净」成立**——修复前 `claude -p` 会偷偷加载本地个人档案，所谓"基线"其实认识读者；切断渗漏后，个性化只走受控的 USER.md→rerank 通道。
- **P3 讲到极致（✅ 大体落地）**：critic 批判层诚实标「哪条可跳过」（换名额不砍名额）+ 详解升级 V4「点燃器」四轴 + 正文抓全（arXiv 全文 + ar5iv 重定向护栏 + 智能截断保住结论段）。**「覆盖更广」一项按 C2 决策反转为收紧**：不加中文源稀释英文前沿、不动源权重——源分布进 WATCHLIST，攒几天干净跑再评估。
- **P4 会聊 + 自进化（🔨 E1 第一步已落地，其余推迟）**：**E1 数据级 reviewer 上线**——`radar --mode review` 每周日 21:00 自动跑：读 eval 趋势 + 我的投票 + top-10 源分布 + 「这条能改进自己」标注 + critic 统计 + WATCHLIST → 产出**观察 + 草案建议**（周报存 `data/self_improve/reviews/`，top-line 摘要自动推钉钉）。**自动的只是「跑和送达」；草案零自动应用，改不改由我拍板。** E 会聊（对话深挖）与 E2 代码级自指闭环仍推迟（E2 别阻塞 E1）。

## 投递与详解的演进（大白话补记）

**投递怎么一路走过来**：最早 = 钉钉群 webhook 发 markdown 简报（单向，只能看）→ Phase A 换成企业机器人 1v1 的**一张互动列表卡**（每行可点 👍/👎，票自动写回 feedback）→ 但完整四轴详解还只躺在本地归档、**从没上过手机** → 投递 v3 把详解渲染成 Cloudflare Pages **阅读页**（URL 按天派生、不可枚举、noindex），卡片每行链到当天页的对应锚点——扫卡+投票在钉钉、点一行进浏览器读全文，零多余消息。

**详解怎么一路走过来**：最早是「论文压缩件」（8 个小节 + 关键数字墙 + 词表）——信息密，但读起来像论文摘要的摘要 → V4 换成「点燃器」：固定四轴（①作者在讲什么 ②为什么重要/反直觉在哪 ③你能怎么用 ④想深挖看这里），砍数字墙只留一两个关键数，整篇 ~2000 字——目标是读完**带走判断**，而不是带走数字。

**三个静默 bug 的教训（含 P0 隐私修复）**：① rerank 三连超时会**静默回退**到便宜模型的分序——个性化根本没跑却不吱声（修：加长超时 + digest 头部降级横幅）；② ar5iv 对未转换的论文 30x 跳回摘要页，4.7K 字也能过闸门**假装拿到了全文**（修：落到 /abs/ 即判失败、改走 PDF）；③ 最大的一个：`claude -p` 会自动加载 cwd 与祖先目录的 CLAUDE.md——每次管线调用都带着两份 gitignore 的个人档案，而 deepread 产出要上**公开**阅读页（修：LLM 子进程 cwd 指向中立目录，祖先链零 CLAUDE.md）。共同教训 = **坏了要吱声**：静默降级/静默污染比失败更危险；每次修完都要真跑自证。

**无人值守落位（2026-07-05）**：仓库从 `~/Desktop` 迁到 `~/agent-radar`（macOS TCC 挡住 launchd 读 Desktop，探针实锤）；launchd 三件套上线——每天 08:30 daily（跑完自动 eval 当天）、serve 常驻接投票（KeepAlive）、每周日 21:00 review。**「每天自动」从这天起字面成立**：每天自动跑 → 我读+投票 → 周度 review 盘点 → 我拍板改进。

> 每个阶段都可独立交付、可演示；每块改动都先做小调研、跑通测试、真跑自证，再推送。
