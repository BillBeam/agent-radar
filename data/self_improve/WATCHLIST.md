# WATCHLIST — 观察项清单（review 每周盘点）

> 这里放「先不动、但要盯」的观察项。每次 `radar --mode review` 会把本清单带进周报盘点。
> 增删观察项 = 用户拍板（连同理由记 `decisions.md`）。观察项不是待办：默认动作是**不动**。

## 1. top-10 源分布（等 2-3 天干净跑再评估）
- **盯什么**：每日 top-10 按 `source_name` 的分布（review 数据段自动给出）。
- **判据/时机**：P0 隐私修复 + rerank 超时修复之后的「干净跑」攒 2-3 天；若某源持续 >50% 或 harness 类源持续缺席，才考虑动源结构。
- **出处**：decisions.md「源结构收口：不调源权重、不砍源、不加多样性配额——来源分布出来后再评估」（07-04 诊断轮）。

## 2. 价值分层（哪篇该精读）的系统化
- **盯什么**：连续读几天 V4 详解后，是否反复出现「10 条里不知道先读哪篇/该跳哪篇」的真实痛感；critic ⚠️可跳过 标注的命中体感。
- **判据/时机**：连续一周日跑后由用户口头判断；痛感确认前不设计、不动轴。
- **出处**：decisions.md C 批判层设计（排序≠过滤、verdict 挂 stats，不污染 tags）；deepread「三档诚实」原则。

## 3. 中段截断与「直接看到结果数字」的需求
- **盯什么**：V4 ②轴「最多留一两个关键数」+ 智能截断（头 70% + 尾 30%）之后，真实阅读中是否需要更多硬数字/被截掉的中段内容。
- **判据/时机**：用几天后由用户判断；若需要，只动截断参数或呈现侧，不回到数字墙。
- **出处**：decisions.md「C 智能截断」（FETCH_CAP 80K→砍尾节→头 70%+尾 30%）+ deepread V4「不甩数字墙」。

## 4. [5] 基线轻微上浮 / 豁免例句一般化
- **盯什么**：rerank.md 豁免清单加「新 benchmark/新测量方法」后，此类条目是否持续上浮挤掉更该上的条目（区分单 run 方差 vs 真漂移）。
- **判据/时机**：多天干净跑对比；07-05 生产首跑护栏已压住「又一个同类基准」，继续观察。
- **出处**：decisions.md「豁免落地——[5] 基线 #3 vs 旧基线 #8 诚实记录为例句轻微一般化，方向无害、幅度在单 run 方差内，观察」。
